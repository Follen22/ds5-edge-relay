# DS5 Edge Relay

**DS5 Edge Relay** — Qt6 GUI-приложение для Linux, которое создаёт виртуальный контроллер DualSense (054C:0CE6) через `/dev/uhid` и проксирует ввод с физического DualSense Edge (054C:0DF2) через `/dev/hidraw*`.

---

## Зачем это нужно

DualSense Edge имеет нестандартный USB PID (0x0DF2) и дополнительные задние кнопки (L4, LB, R4, RB, LFN, RFN), которые Proton и многие Linux-игры не распознают. Relay-демон заставляет Edge выглядеть как стандартный DualSense — восстанавливая совместимость с играми и добавляя переназначение кнопок.

---

## Возможности

| Функция | Описание |
|---|---|
| HID-ретрансляция | Читает сырые репорты с физического Edge, пересылает на виртуальный DS5 |
| Переназначение кнопок | Визуальный редактор — кликай на геймпад, чтобы создать бинды |
| Мультинаправленный D-pad | Бинды на несколько направлений крестовины правильно объединяются в диагонали |
| Живое обновление биндов | Включай/выключай бинды прямо во время работы — перезапуск не нужен |
| Эксклюзивные кнопки Edge | LFN, RFN, LB, RB доступны как источники для переназначения |
| Обратная связь | Haptics, адаптивные триггеры и LED-команды пересылаются обратно на устройство |
| Автоподключение | Автоматически переподключается после отсоединения и повторного подключения |
| Системный трей | Сворачивается в трей; «Работать в фоне» сохраняет релей после закрытия окна |
| Автозапуск | Одним кликом добавляет/убирает запись в `~/.config/autostart/` |
| Локализация | Русский / Английский интерфейс, переключается на лету |
| Кастомный тайтлбар | Безрамочное окно с нативными кнопками в стиле приложения |

## Требования

- **ОС:** Linux (рекомендуется ядро ≥ 5.15)
- **Runtime:** Qt 6.2+
- **Сборка:** CMake ≥ 3.20, Ninja, компилятор C++23
- **Устройства:** `/dev/uhid`, `/dev/hidraw*`

## Установка

### Из AUR (Arch Linux)

```bash
yay -S ds5-edge-relay
# или
paru -S ds5-edge-relay
```

### Ручная сборка

```bash
git clone https://github.com/Follen22/ds5-edge-relay
cd ds5-edge-relay
cmake -B build -GNinja -DCMAKE_BUILD_TYPE=Release
ninja -C build
sudo ninja -C build install   # опционально
```

### Права на устройства (без sudo)

Пакет автоматически устанавливает udev-правила. Чтобы применить немедленно:

```bash
sudo udevadm control --reload-rules
sudo udevadm trigger --subsystem-match=hidraw
```

Добавьте себя в группу `input` для надёжного доступа:

```bash
sudo usermod -aG input $USER
# затем выйдите и войдите, или:
newgrp input
```

## Использование

### Запуск ретрансляции

1. Запустите **DS5 Edge Relay** из меню приложений или командой `ds5-edge-relay`.
2. Подключите DualSense Edge по USB или Bluetooth.
3. Нажмите **▶ Старт** — индикатор статуса станет зелёным, виртуальный контроллер появится в системе.
4. Запустите игру. Она увидит стандартный DualSense (054C:0CE6).
5. Нажмите **⏹ Стоп** для отключения виртуального контроллера.

### Переназначение кнопок

1. Поставьте галочку **Использовать бинды** — раскроется редактор.
2. Нажмите **Добавить бинд**.
3. Кликните **кнопку-триггер** на схеме геймпада (кнопку, которую хотите переназначить).
4. Кликните одну или несколько **целевых кнопок** (что должно нажиматься). Повторный клик снимает выбор.
5. Нажмите **Применить** — бинд появится в списке.
6. У каждого бинда есть чекбокс для включения/выключения **на лету** (вступает в силу при следующем HID-репорте).
7. Нажмите **✕** рядом с биндом для его удаления.

Бинды сохраняются в `~/.config/ds5-edge-relay/binds.json` и загружаются автоматически при следующем запуске.

**Особенности D-pad:**
- Бинд кнопки на несколько направлений крестовины создаёт диагонали (например, Вверх+Вправо → СВ).
- Противоположные направления взаимно отменяют друг друга (Влево+Вправо → нейтраль, Влево+Вверх+Вправо → Вверх).

### Работа в фоне

Включите **Работать в фоне при закрытии окна** — ретрансляция продолжает работать в системном трее после закрытия окна. Кликните иконку в трее, чтобы открыть снова.

### Автозапуск вместе с системой

Включите **Автозапуск вместе с системой**, чтобы добавить XDG-запись автозапуска. Приложение будет стартовать автоматически при входе.

Альтернативно, включите systemd user-сервис:

```bash
systemctl --user enable --now ds5-edge-relay
```

## Формат файла биндов

`~/.config/ds5-edge-relay/binds.json`

```json
{
  "binds": [
    { "enabled": true, "trigger": "LB", "actions": ["Cross"] },
    { "enabled": true, "trigger": "RB", "actions": ["DPadUp", "DPadRight"] }
  ]
}
```

Допустимые имена кнопок: `Cross`, `Circle`, `Square`, `Triangle`, `L1`, `R1`, `L2`, `R2`, `L3`, `R3`, `Options`, `Create`, `PS`, `Touchpad`, `DPadUp`, `DPadDown`, `DPadLeft`, `DPadRight`, `LFN`, `RFN`, `LB`, `RB`.

## Архитектура

```
MainWindow (Qt GUI thread)
├── BindEditorWidget   — визуальный редактор биндов, эмитит bindingsChanged()
│   ├── GamepadWidget  — кликабельная схема геймпада (1.png, режим Screen)
│   └── BindStorage    — JSON-хранилище (~/.config/ds5-edge-relay/binds.json)
└── RelayWorker (QThread)
    ├── find_hidraw_device()  — сканирует sysfs в поиске VID:PID 054C:0DF2
    ├── UhidDevice            — создаёт /dev/uhid виртуальный DualSense 054C:0CE6
    ├── poll() loop           — мультиплексирует физический hidraw ↔ виртуальный uhid
    ├── apply_bindings()      — переназначает кнопки в буфере HID-репорта
    └── update_bindings()     — потокобезопасное живое обновление из UI-потока
```
